# Data inputs

The release ships an empty `data/` skeleton. Everything in this folder is generated locally, no datasets are bundled.

```
data/
  population/        synthetic population (must be provided)
  geo/               district polygon (optional, for the map dashboard)
  raw/               raw influenza surveillance data (optional, for calibration)
  results/           run outputs (created at runtime)
```

## Synthetic population

The GABM behavioural layer needs a synthetic-population file at:

```
data/population/households.txt
```

### Format

Tab-separated, one row per agent. Required columns:

| Column | Type | Meaning |
|---|---|---|
| `sp_id` | int | Stable per-agent id. |
| `hh_id` | int | Household id (every agent belongs to exactly one). |
| `latitude` | float | WGS84 latitude. |
| `longitude` | float | WGS84 longitude. |
| `age` | int | Agent age in years. |
| `sex` | str | `M` / `F`. |
| `archetype` | str | One of `child`, `young_active`, `middle_family`, `service_worker`, `healthcare`, `elderly`. |
| `workplace_id` | int? | Workplace id (`-1` if retired / unemployed). |
| `school_id` | int? | School id (`-1` if not in school). |

The loader (`src/utils/data_loader.py`) reads this file and builds the per-day contact network (household, workplace, school, casual).

### Where to obtain a compatible population

The thesis uses the synthetic population of Vasileostrovsky district (Saint Petersburg) generated by the Leonenko lab and released alongside:

> Leonenko V. et al. *Influenza dynamics in a synthetic population.* Journal of Computational Science 41 (2020).

Any synthetic population that follows the column schema above will work. The pipeline does not depend on a specific city.

## Geo file (optional)

For the Streamlit dashboard's map view:

```
data/geo/vasileostrovsky.geojson
```

A single GeoJSON polygon describing the district boundary. The map zooms to this polygon and clips agent markers to it.

If you have a different region, swap the file and update `data.geo_file` in the YAML config (see `CONFIG.md`).

## Raw surveillance data (optional, for calibration)

If running the ABC calibration experiments (`exp8_calibration_forecast.py`, `exp14_forecasting_smc.py`):

```
data/raw/
  influenza_2014-2015.csv
  influenza_2015-2016.csv
  ...
```

CSV format: one row per week, columns `week`, `incidence`, `population`. The calibration season is set via `data.calibration_season` in the YAML config.

## Results directory

Created at runtime by every entry point. Each run writes a fresh subdirectory under:

```
data/results/
  <run-id>/
    metrics.json
    snapshots/
      day_001.json
      day_002.json
      ...
    logs/
      prompts.jsonl
      responses.jsonl
    figures/
      *.png
```

`run-id` is `<config-name>_<backend>_<seed>_<timestamp>`.

## What is NOT in this release

The release folder intentionally contains no data, no figures, no thesis text, and no paper drafts. Everything in `data/` is generated locally from the synthetic population you provide.
